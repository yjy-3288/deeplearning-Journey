从数学的角度看，有监督的学习算法就是一个寻找映射函数 $f$ 的过程(监督学习是一个大框架,可以采取多种形式的模型)。假设有输入数据 $X$（特征）和对应的标签 $Y$（答案）。有监督学习算法的目标就是利用训练数据，找到一个函数：

$$Y=f(X)$$

一旦这个 $f$ 被找出来了（也就是模型训练好了），当你给它一个从未见过的 $X_{new}$，它就能通过这个函数计算出预测值 $\hat{Y}$。

**为什么不用全梯度下降？**

1000 个一起上（全批量）：它是极其稳健的。每一次更新的方向都绝对指向当前最完美的“下坡”方向。
缺陷：如果山坡上有一个小坑（局部最小值），全批量由于太听话，会直接滑进坑里死掉，再也出不来了。

10 个一组（小批量）：每一组 10 道题给出的建议其实是带点偏见的。这组 10 道题可能让你往左偏，下一组 10 道题可能让你往右偏。

优势：这种“摇摇晃晃”的下山路径自带抖动。这种抖动往往能让模型在滑进小坑时，被随机的噪声“踢”出来，从而有机会找到真正的山谷（全局最小值）。

在训练的数据当中,那个"label"是加过噪声的,而这个噪声服从正态分布(现实中的噪声往往是由无数个微小、独立且随机的干扰因素叠加而成的，而根据中心极限定理，这种“大规模混乱的累加”在宏观上必然会趋向于完美的钟形正态分布)

**用集合映射的语言来精确描述“softmax回归的输出层是一个全连接层”**

从 $\mathbb{R}^4$ 到 $\mathbb{R}^3$ 的仿射映射：$`f(\mathbf{x}) = \mathbf{W}\mathbf{x} + \mathbf{b}`$

**是满射**

只要这个矩阵的行向量是线性无关的（即行满秩，这在随机初始化的神经网络中发生概率是 100%），这个仿射变换就能覆盖整个 3 维输出空间 $\mathbb{R}^3$
  
**proof:不是单射**

lemma:

已知

$$\dim(\text{输入空间}) = \text{rank}(\mathbf{W}) + \dim(\ker(\mathbf{W}))$$

又因$`\text{rank}(\mathbf{W}) \leq 3`$,
故$`\dim(\ker(\mathbf{W})) \geq 1`$

现证明: 如果 $`\mathbf{x}_1 \neq \mathbf{x}_2`$，则必定 $`f(\mathbf{x}_1) \neq f\mathbf{x}_2)`$

假设有一个输入向量 $\mathbf{x}$，它映射到了输出 $\mathbf{y}$：

$$\mathbf{W}\mathbf{x}+\mathbf{b}=\mathbf{y}$$

因为我们刚才证明了 $\dim(\ker(\mathbf{W})) \geq 1$，所以一定存在一个非零向量 $`\mathbf{v}`$ 属于这个核（即 $`\mathbf{W}\mathbf{v} = \mathbf{0}`$ 且 $`\mathbf{v} \neq \mathbf{0}`$）。现在，我们构造一个新的输入点：$`\mathbf{x}_{new} = \mathbf{x} + \mathbf{v}`$。显然，$`\mathbf{x}_{new} \neq \mathbf{x}`$。但如果我们把 $`\mathbf{x}_{new}`$ 喂给全连接层：

$$\begin{aligned} f(\mathbf{x}_{new}) &= \mathbf{W}(\mathbf{x} + \mathbf{v}) + \mathbf{b} \\ 
&= \mathbf{W}\mathbf{x} + \mathbf{W}\mathbf{v} + \mathbf{b} \\
&= \mathbf{W}\mathbf{x} + \mathbf{0} + \mathbf{b} \\ 
&= \mathbf{y} \end{aligned}$$

证毕！ 

---
小数连乘会导致数值下溢出,故考虑对数似然,变成连加。极大似然估计的目标就是找到一个参数$`W^*`$使得对数似然函数达到最大,等价于最小化负对数似然(交叉熵损失),而方程没有解析解,因此需要求导算梯度找最优参数.

**为什么交叉熵损失先对$`o_j`$ 求导？（追责机制）**

$$ W \stackrel{\text{计算}}{\longrightarrow} o_j \stackrel{\text{Softmax}}{\longrightarrow} \hat{y}_j \stackrel{\text{交叉熵}}{\longrightarrow} l $$

$$\frac{\partial l}{\partial \mathbf{w}_j} = \frac{\partial l}{\partial o_j} \cdot \frac{\partial o_j}{\partial \mathbf{w}_j} = (\hat{y}_j - y_j) \cdot \mathbf{x}$$

偏导数衡量的是“敏感度”:

如果在反向调查时，发现稍微动一下员工甲的参数，最终的 Loss 就会产生剧烈的波动，说明员工甲就是影响全局的关键节点（根源）。就会被贴上一张数值极大的梯度条子。
如果动一下员工乙的参数，Loss 根本没什么反应，说明员工乙的错误无关紧要，梯度就会很小.

```
y = torch.tensor([0, 2])
y_hat = torch.tensor([[0.1, 0.3, 0.6], [0.3, 0.2, 0.5]])
y_hat[[0, 1], y]
```
y表示,在这一批次的训练数据中,只抽了两张照片,其正确答案分别是0=狗和2=猫。而y_hat则表示样本1和样本2在三个类别(0=狗,1=猪,2=猫)中分别作预测得到的概率。而每一个训练数据中的样本都应该有其正确答案,y_hat[[0, 1], y]则是找到每个样本在此次训练之下,给"标准答案"作出的预测概率。

交叉熵公式:

$$L = - \sum_{j=1}^q y_j \log(\hat{y}_j)$$

假设有一个 3 分类问题，某张图片的真实标签是类别 2=猫。它的真实概率分布 $y$ 写成独热编码就是：$`y = [0, 0, 1]`$。模型的预测概率是：$`\hat{y} = [0.1, 0.8, 0.1]`$。现在，开始套公式算交叉熵：

$$L = - (y_0 \log\hat{y}_0 + y_1 \log\hat{y}_1 + y_2 \log\hat{y}_2)$$

代入数字：

$$L = - (0 \times \log 0.1 + 0 \times \log 0.8 + 1 \times \log 0.1)$$

所以,只需要一行代码可以实现交叉熵损失函数:
```
def cross_entropy(y_hat, y):
return- torch.log(y_hat[range(len(y_hat)), y])
cross_entropy(y_hat, y)
```
此时输出```tensor([2.306,0.6931])```,也就是根据交叉熵损失函数打的分.

当然,可以一次性抽4张照片,或者更多:```y = torch.tensor([0, 2, 0, 1])```。此时打的分数可以是(打比方)```tensor([2.306,0.6931,0.7891,4.1565])```,它里面的每一个数字，都和最初的图片有着极其严密的物理绑定关系

**训练**

定义一个函数来训练一个迭代周期
```
def train_epoch_ch3(net, train_iter, loss, updater): #@save
    """训练模型一个迭代周期（定义见第3章）"""
    # 将模型设置为训练模式
    if isinstance(net, torch.nn.Module):
        net.train()
    # 训练损失总和、训练准确度总和、样本数
    metric = Accumulator(3)
    for X, y in train_iter:
        # 计算梯度并更新参数
        y_hat = net(X)
        l = loss(y_hat, y)
        if isinstance(updater, torch.optim.Optimizer):
            # 使用PyTorch内置的优化器和损失函数
            # grad采用累加赋值,
            updater.zero_grad()
            l.mean().backward()
            updater.step()
        else:
            # 使用定制的优化器和损失函数
            l.sum().backward()
            updater(X.shape[0])
        metric.add(float(l.sum()), accuracy(y_hat, y), y.numel())
    # 返回训练损失和训练精度
    return metric[0] / metric[2], metric[1] / metric[2]
```
训练循环实际上是这样的：

- 拿 32 张图 喂进流水线。

- 图片流过 net[0], net[1], net[2]... 最终吐出预测结果。

- 算 Loss。backward()：从后往前跑，给每一层（net[0], net[1]...）的参数口袋里都塞进梯度。

- updater.step()：优化器查看名册，发现名册里有 net[0] 的 $`w,b`$，也有 net[1] 的参数... 于是一把抓，把所有层的参数全都更新了。

- 清空所有层的梯度，迎接下 32 张图。


注: 参数的更新过程如下

$$\mathbf{w}_i^{(\text{new})} = \mathbf{w}_i^{(\text{old})} - \eta \frac{\partial L}{\partial \mathbf{w}_i}$$

$$\mathbf{w}_i^{(\text{new})} = \mathbf{w}_i^{(\text{old})} - \eta (\hat{y}_i - y_i) \mathbf{x}$$

**问题:在书中介绍的softmax的net只连接了两层,如果是多层,该如何反向传播(计算梯度,更新参数)？**

假设我们有一个精简版的 3 层神经网络。我们忽略偏置项，用最简单的公式表示它的前向传播（从左到右算答案）：

- 第 1 层： $`A_1 = \sigma(W_1 X)`$   (输入数据 $X$，算出第 1 层输出 $A_1$)
- 第 2 层： $`A_2 = \sigma(W_2 A_1)`$  (输入 $A_1$，算出第 2 层输出 $A_2$)
- 第 3 层： $`\hat{Y} = W_3 A_2`$       (输入 $`A_2`$，算出最终预测结果 $`\hat{Y}`$)最终误差： $`L = \text{Loss}(\hat{Y}, Y)`$

根据微积分链式法则，既然 $W_2$ 影响了 $`A_2`$，$A_2$ 影响了 $`\hat{Y}`$，$`\hat{Y}`$ 影响了 $L$，我们就把这根链条从右往左反着乘起来：

$$\frac{\partial L}{\partial W_2} = \underbrace{ \frac{\partial L}{\partial \hat{Y}} \cdot \frac{\partial \hat{Y}}{\partial A_2} }_{\text{来自第3层的误差信号}} \cdot \underbrace{ \frac{\partial A_2}{\partial W_2} }_{\text{第2层本身的局部导数}}$$

第 2 层在算梯度时，不仅需要上面传下来的 $`\delta_3`$，还需要自己前向传播时的输入数据 $A_1$。这意味着，在数据从左到右流动（做题）的时候，神经网络不能算完即扔！它必须把每一层中间产生的临时结果（激活值）全部像宝藏一样保存在显存里，一直等到反向传播时拿出来用。

这就是为什么层数越多（比如 100 层的 ResNet），训练时我们的显存（VRAM）就越容易爆掉的原因——它在为了反向推导做准备。

**2. 为什么梯度可以累加？**

$$\frac{\partial L_{\text{total}}}{\partial w} = \frac{1}{n} \left( \frac{\partial l_1}{\partial w} + \frac{\partial l_2}{\partial w} + \dots + \frac{\partial l_{n}}{\partial w} \right)$$

在多层感知机中，权重成为了一个矩阵

$`\mathbf{X}`$ (输入数据)： 形状是 $`n \times d`$。（$`n`$ 套房子，$`d`$ 个基础特征）

$`\mathbf{W}^{(1)}`$ (隐藏层权重)： 形状是 $`d \times h`$。（$`d`$ 个基础特征映射到 $`h`$ 个高级特征）

当它们相乘时（$`\mathbf{X}\mathbf{W}^{(1)}`$）：根据矩阵乘法规则（中间的 $`d`$ 抵消），得到的新矩阵 $`\mathbf{H}`$ 形状必然是 $`n \times h`$。$`\mathbf{H}`$ (隐藏层输出)： 形状是 $`n \times h`$。（$`n`$ 套房子，每套房子现在被提取出了 $`h`$ 个高级特征）

**什么是模型泛化？**

泛化，就是模型在“没见过的新数据”上的表现能力。

**什么会导致（或影响）泛化能力？**

泛化能力的好坏，通常在两个极端之间摇摆：欠拟合（学得太少）和过拟合（背得太死）。导致泛化能力变化的核心原因有以下几个：

- 模型的复杂度（脑容量）：

  - 太简单（欠拟合）： 就像让小学生去做微积分，脑容量不够，连练习册（训练集）都做不及格，更别提期末考试（测试集）了。泛化极差。

  - 太复杂（过拟合）： 模型的参数太多，像一个拥有超级记忆力的学生。他不去理解公式，而是直接把练习册上所有题目的答案（甚至是印刷错误）全部死记硬背下来。平时测验 100 分，一到真正的高考（遇到新题）直接崩溃。泛化也极差。

- 数据的质量和数量：

  - 如果数据太少，或者全是极其相似的数据（比如全是白猫），模型就无法总结出普遍规律。数据越多、越多样（黑猫、白猫、胖猫、瘦猫），模型的泛化能力就越强。

- 训练时间：

  - 训练得太久，模型在把有用的规律学完之后，就会开始去记数据里的“噪音”（比如照片上的水印）。这就好比学生考前焦虑，连课本的页码都背下来了，反而影响正常发挥。

**泛化会带来什么影响？**
- 泛化好带来的影响：比如，泛化好的自动驾驶模型，不仅能在测试场里开得好，还能在它从来没去过的真实街道上安全避让行人。它意味着 AI 真正具备了“智能”，可以大规模部署到现实世界中赚钱。

- 泛化差带来的影响： 模型在实验室里各项指标完美（准确率 99%），一旦上线投入真实环境就疯狂报错（准确率暴跌到 50%）。这会导致极其恶劣的用户体验、资金损失，甚至安全事故。

**训练集-验证集-测试集**

验证集和测试集都是在参数固定后给模型，然后得到分数。区别仅在于，验证集得到的结果不代表最终表现，其还需要继续训练

这三个集合分别对应:训练误差,验证误差,测试误差.而测试误差是人类用来估算泛化误差的最佳手段

暂退法Dropout只用于训练期间

下面解释过程:

**进入隐藏层 1 (线性计算 + 激活)**
```
H1 = self.relu(self.lin1(X.reshape((-1, self.num_inputs))))
```
784 个原始像素特征，首先经过第一层网络（256 个神经元）的加权计算，然后通过 ReLU 激活函数(把负数全变成 0).此时,这 256 个神经元刚刚算出了第一波“初级特征”（比如衣服的边缘、轮廓）

-1代表自动推断:送了 1 张图，X 就是 [1, 784] 的矩阵（行向量）;送了 256 张图，X 瞬间变成 [256, 784] 的巨大矩阵

**第一波 Dropout(20% 成为0)**
```
if self.training == True:
    H1 = dropout_layer(H1, dropout1)  # dropout1 = 0.2
```
按照 20% 的概率，这 256 个值里大约有 50 个会瞬间变成 0，剩下的 200 多个值会被强行放大（除以 0.8）

**进入隐藏层 2 (带着残缺的记忆继续算)**
```
H2 = self.relu(self.lin2(H1))
```
第二层网络（又是 256 个神经元）开始接收上一层的数据.此时,因为上一层的 H1 里有 20% 是 0，这意味着第二层的神经元在做计算时，接收不到20%的信息。所以只能根据剩下的 80% 的特征去拼凑、猜测这到底是一件什么衣服。这就逼迫第二层神经元绝不能依赖第一层的某几个“学霸”，必须自己多学点本事。算完后通过 ReLU 激活，得到高级特征 H2。

**第二波 Dropout**
```
if self.training == True:
    H2 = dropout_layer(H2, dropout2)  # dropout2 = 0.5
```
大约 128 个神经元输出变 0，剩下的放大 2 倍。

**输出层 (做出最终决定)**
```
out = self.lin3(H2)
return out
```
最后一层（10 个神经元，代表 10 个服装类别）接收到的是被砍掉了一半的 H2。它在这极度残缺的信号中，被迫做出最后的分类预测（算出 10 个类别的得分）。

注:不要在输出层后面加Dropout

**方差可以被理解为:"信号强度"**

因为方差衡量的是一组信息的波动程度，而波动本身就是信息.一张图片（比如经典的 MNIST 手写数字，大小是 28x28）并不是一个数字，而是整整 784 个像素值（特征）排成的一长串队伍.对于单张图片来说，特征方差 = 图像的对比度(信息丰富度).纯黑的图片，方差为0; 而色彩斑斓的风景照,方差则较大.

**为什么要“输出的方差 = 输入的方差”？**

假设我们有一个 100 层的深层神经网络，也就是一个由 100 个信号中继站组成的超长通讯线路。你站在起点（第 1 层），对着麦克风喊了一句话（输入图像 $X$）。

- 情况1（输出方差 < 输入方差）：假设我们初始化的权重不好，导致信号每经过一层，方差（能量）都会衰减 10%，变成原来的 0.9。经过 100 层后，信号能量变成了 $0.9^{100} \approx 0.000026$。结果： 传到第 100 层输出端时，声音已经变成了比蚊子还小的杂音。最后一层的神经元什么都听不见,这就是前向传播的信号消失（特征丢失）。

- 情况2（输出方差 > 输入方差）：假设我们把权重初始化得太大了，每经过一层，信号被放大了 10%，变成原来的 1.1。经过 100 层后，信号能量变成了 $1.1^{100} \approx 13780$。结果： 传到第 100 层时，信号变得震耳欲聋。数值大到超出了计算机能表示的极限（溢出），所有神经元都被震瘫痪了，输出全变成了 NaN（不是一个数字）。

因此,要求输入输出的数据之间，保持相同的特征对比度。

而为了让输入方差=输出方差,有下面定理.也就是Xavier初始化的原理

---
Xavier初始化是一种初始化权重参数的方法
**定理**

考虑全连接神经网络中的任意相邻两层。设输入向量为 $`\mathbf{x} \in \mathbb{R}^{n_{\text{in}}}`$，权重矩阵为 $`\mathbf{W} \in \mathbb{R}^{n_{\text{out}} \times n_{\text{in}}}`$，无偏置项，线性输出向量为 $`\mathbf{y} \in \mathbb{R}^{n_{\text{out}}}`$，即 $`\mathbf{y} = \mathbf{W}\mathbf{x}`$。为满足前向传播的方差守恒假设（即 $`\forall i, j, \text{Var}(y_i) = \text{Var}(x_j)`$），权重矩阵 $`\mathbf{W}`$ 的元素 $`w_{ij}`$ 必须从方差为 $`\frac{1}{n_{\text{in}}}`$ 的独立同分布（i.i.d.）中抽取。

**为使证明成立，我们建立以下严格概率论假设：**

1.输入的独立同分布 (i.i.d.)：输入向量的各分量 $`x_1, x_2, \dots, x_{n_{\text{in}}}`$ 是独立同分布的随机变量，且满足 $`\mathbb{E}[x_j] = 0`$，方差恒定 $`\text{Var}(x_j) = \sigma_x^2`$。

2.权重的独立同分布 (i.i.d.)：权重矩阵的各元素 $`w_{ij}`$ 是独立同分布的随机变量，且满足 $`\mathbb{E}[w_{ij}] = 0`$，方差恒定 $`\text{Var}(w_{ij}) = \sigma_w^2`$。（此处引入命题核心）

3.互独立性：输入向量 $`\mathbf{x}`$ 与权重矩阵 $`\mathbf{W}`$ 彼此完全独立。

**Proof**

考察输出向量 $`\mathbf{y}`$ 的第 $`i`$ 个分量 $`y_i`$，其代数表达式为：

$$y_i = \sum_{j=1}^{n_{\text{in}}} w_{ij} x_j$$

对等式两边取方差运算（Variance）：

$$\text{Var}(y_i) = \text{Var}\left( \sum_{j=1}^{n_{\text{in}}} w_{ij} x_j \right)$$

根据假设 1、2 和 3，对于不同的 $`j`$，乘积项 $`(w_{ij} x_j)`$ 彼此相互独立。根据概率论中“独立随机变量和的方差等于方差的和”之引理，求和符号与方差算子可交换：

$$\text{Var}(y_i) = \sum_{j=1}^{n_{\text{in}}} \text{Var}(w_{ij} x_j)$$

接下来，展开单个乘积项的方差 $`\text{Var}(w_{ij} x_j)`$。根据方差定义 $`\text{Var}(Z) = \mathbb{E}[Z^2] - (\mathbb{E}[Z])^2`$：

$$\text{Var}(w_{ij} x_j) = \mathbb{E}[(w_{ij} x_j)^2] - (\mathbb{E}[w_{ij} x_j])^2$$

根据假设 3（$`\mathbf{x}`$ 与 $`\mathbf{W}`$ 独立），两个独立随机变量乘积的期望等于期望的乘积：

$$\mathbb{E}[w_{ij} x_j] = \mathbb{E}[w_{ij}]\mathbb{E}[x_j]$$

由假设 1 和 2（均值皆为 0），上式为：

$$\mathbb{E}[w_{ij}]\mathbb{E}[x_j] = 0 \cdot 0 = 0$$

同样根据独立性，处理平方项的期望 $`\mathbb{E}[(w_{ij} x_j)^2]`$：

$$\mathbb{E}[w_{ij}^2 x_j^2] = \mathbb{E}[w_{ij}^2]\mathbb{E}[x_j^2]$$

由于均值为 0，二阶原点矩即等于方差，即 $`\mathbb{E}[w_{ij}^2] = \text{Var}(w_{ij}) = \sigma_w^2`$，且 $`\mathbb{E}[x_j^2] = \text{Var}(x_j) = \sigma_x^2`$。将其代入单项方差等式：

$$\text{Var}(w_{ij} x_j) = \sigma_w^2 \sigma_x^2 - 0 = \sigma_w^2 \sigma_x^2$$

将此结果代回 $`y_i`$ 的总方差求和公式。注意：正是因为假设 1 和 2 规定了 $`x_j`$ 之间同分布、 $`w_{ij}`$ 之间同分布，$`\sigma_w^2 \sigma_x^2`$ 才成为不依赖于下标 $`j`$ 的常数：

$$\text{Var}(y_i) = \sum_{j=1}^{n_{\text{in}}} (\sigma_w^2 \sigma_x^2) = n_{\text{in}} \sigma_w^2 \sigma_x^2$$

根据我们的终极目标“前向方差守恒”，要求 $`\text{Var}(y_i) = \text{Var}(x_j)`$，即：

$$\sigma_x^2 = n_{\text{in}} \sigma_w^2 \sigma_x^2$$

假设输入信号存在有效信息（$`\sigma_x^2 \neq 0`$），等式两边同时消去 $`\sigma_x^2`$：

$$1 = n_{\text{in}} \sigma_w^2$$

求解得出权重方差的唯一必要条件：

$$\text{Var}(w_{ij}) = \sigma_w^2 = \frac{1}{n_{\text{in}}}$$

Q.E.D.

**困境**

为了让前向传播时信号不消失不爆炸，权重的方差必须是 $`\sigma^2 = \frac{1}{n_{in}}`$.但为了让反向传播时梯度不消失不爆炸，权重的方差又必须是 $`\sigma^2 = \frac{1}{n_{out}}`$ 

既然不能选 $`\frac{1}{n_{in}}`$ 也不能选 $`\frac{1}{n_{out}}`$，那就把它们两个的需求加起来求个平均！

作者让前向和反向的节点数共同决定方差：

$$\frac{n_{in} + n_{out}}{2} \sigma^2 = 1$$

解出来，就是现代深度学习框架里默认的标准方差公式：

$$\sigma^2 = \frac{2}{n_{in} + n_{out}}$$

**分布漂移**

简单来说就是:现实世界是动态的。当我们把实验室里训练好的完美模型放到真实世界去跑时，数据的环境、来源、甚至定义都可能发生改变。

**数据从某个分布中采样,是什么意思？**

去街头拍 10000 张猫的照片。这 10000 张照片，就是从“猫的真实分布”中采样（Sample）出来的结果。
我们假设，如果采样的数量足够多，并且是随机抽的（独立同分布，i.i.d.），那么这 10000 张照片就能近似代表全宇宙猫的规律。

**训练数据和测试数据的分布为什么要有相互关系？**

为了把旧知识迁移到新环境里。比如今天是晴天，明天是阴天，而训练数据是用晴天照片训练的，如果想要迁移到阴天，也能够很好地作出预测，那么就需要假设，核心的因果规律不会变

**协变量漂移**

在机器学习里，我们通常把输入的数据叫作特征（Features），记作 $`\mathbf{X}`$。
但在传统的统计学和概率论里，统计学家更喜欢叫它协变量（Covariates）

在概率论中，我们描述一个系统时，最完整的信息就是它的联合概率分布 $`P(\mathbf{X}, Y)`$。根据条件概率公式，我们可以把它拆解成两部分相乘：

$$P(\mathbf{X}, Y) = P(Y|\mathbf{X}) \cdot P(\mathbf{X})$$

“协变量偏移”在数学上的严格定义，就是指在这两个拆解出来的部分中，一个变了，另一个绝对没变：

- 变了的是 $`P(\mathbf{X})`$（边缘概率分布偏移）：训练集里的数据分布 $`P_{train}(\mathbf{X})`$，和测试集里的数据分布 $`P_{test}(\mathbf{X})`$ 不相等。也就是说，特征出现的频率、区间或环境发生了改变。

- 绝对没变的是 $`P(Y|\mathbf{X})`$（条件概率分布不变）：给定某个特定的特征 $`\mathbf{X}`$ 时，它对应输出 $`Y`$ 的物理规律或判定标准是永恒不变的。即 $`P_{train}(Y|\mathbf{X}) = P_{test}(Y|\mathbf{X})`$。

**纠正**

有原分布$`q(\mathbf{x})`$和目标分布$`p(\mathbf{x})`$.如果训练数据$`q(\mathbf{x})`$的比例是"错误"的,又无法去收集目标数据来训练,这就造成了"漂移",而解决办法可以是:

$$\text{权重 } \beta_i = \frac{p(\mathbf{x}_i)}{q(\mathbf{x}_i)}$$

- 对于一张“阳光大道”的照片： 它在未来哈尔滨出现的概率 $p(\mathbf{x})$ 很小，但在加州训练集 $q(\mathbf{x})$ 里极多。算出来的权重 $\beta$ 可能只有 0.01。模型在训练时几乎会无视这张照片。
- 对于一张“冰雪路面”的照片： 它在哈尔滨 $p(\mathbf{x})$ 是绝对主力，但在加州 $q(\mathbf{x})$ 里极其罕见。算出来的权重 $\beta$ 可能高达 80。这就相当于给这张极其稀缺的照片加上了高音喇叭。模型为了不被扣分，会拼命去学习这张照片里的特征

有了权重之后,使用"加权经验(训练集)风险(损失)最小化"来训练模型:

$$\text{minimize} \frac{1}{n} \sum_{i=1}^{n} \beta_i \cdot l(f(\mathbf{x}_i), y_i)$$

通过给不同样本分配不同的 $`\beta_i`$ 惩罚倍数，强迫模型在训练时“偏心”于未来世界中更重要的样本.

**为什么需要引入 $z$ 标签与混合数据集？**

在纠正协变量偏移时，我们需要计算样本的重要性权重：$`\beta = \frac{p(\mathbf{x})}{q(\mathbf{x})}`$
（其中 $`p(\mathbf{x})`$ 为测试集密度，$`q(\mathbf{x})`$ 为训练集密度）

- 现实困境:
直接计算高维复杂数据（如图像特征）的概率密度 $`p(\mathbf{x})`$ 和 $`q(\mathbf{x})`$，在数学和计算上几乎是不可能完成的任务

- 破局思路:
我们就把“机器不擅长的密度估算问题”，伪装成“机器擅长的二元分类问题”。

   - 强行剥离原始标签。给训练集打上 $`z = -1`$，给测试集打上 $`z = 1`$。
   - 将两者混装，训练一个二元分类器（如逻辑回归），让它去猜某条数据“属于哪个集合”。


根据贝叶斯定理，这个分类器输出的概率比值，在数学上等价于我们想求的密度比值：

$$\beta = \frac{p(\mathbf{x})}{q(\mathbf{x})} = \frac{P(z=1|\mathbf{x})}{P(z=-1|\mathbf{x})}$$

总结：
> 想要训练一个具有协变量偏移纠正过程的模型，其内在的h，相当于是模型里嵌套需要训练的一个小模型，其非常简单，是做线性预测or最开始书中提到的二元分类的，而训练数据也是通过Z标签简化后的版本，而h需要做的任务就是通过训练来得到:输入特征$`\mathbf{x}_i`$属于集合{Z=1}的'打分',这个分数可以被转化成其在后续训练中的"重要性权重$`\beta_i`$"

**机器学习训练架构对比：标准 ERM vs 协变量偏移纠正**

在面对未来的测试环境改变（协变量偏移）时，传统的训练方法会失效，我们需要引入“两段式”的加权训练法。

**1. 标准经验风险最小化 (Standard ERM)**

这是最常规的深度学习训练方式。

- 核心假设:
“过去等于未来（训练集和测试集分布一致）”。

- 数据输入:
训练集的特征 $X$ 和真实标签 $Y$。

- 训练主角:
只有唯一的主任务模型 $f$（如 ResNet）。

- Loss 计算:
人人平等，所有样本的权重都是 1。直接求平均损失：

$$\text{Loss} = \frac{1}{n} \sum_{i=1}^{n} l(f(x_i), y_i)$$

- 弱点:
一旦测试集的分布发生偏移，模型会因为过度拟合过去的“多数派”而在未来惨败。

**2. 协变量偏移纠正 (Covariate Shift Correction)**

这种方法并没有推翻传统方法，而是在传统方法前面强行插入了一个“情报收集”阶段，提前算出未来哪些数据更值钱。

**阶段一：计算样本权重 $`\beta`$**

- 扔掉真实标签 $`Y`$。把训练集特征标为 $`z=-1`$，测试集特征标为 $`z=1`$，全部混在一起。

- 训练一个分类器 $`h`$，让它区分数据来自哪个集合。

- 通过分类器 $`h`$ 的原始打分，给训练集里的每个样本结算出一个固定的身价权重：

$$\beta_i = \exp(h(\mathbf{x}_i))$$

**关于$`h(x)`$**

如果使用逻辑回归:

$$h(\mathbf{x}) = \mathbf{w}^T\mathbf{x} + b$$

其训练(参数更新过程)使用的就是一开始书中介绍的二元分类问题or线性预测问题。在满足了loss的制定标准之后，其输出就是一个数字，而需要通过sigmoid函数来压缩成0-1区间上的概率

$$P(z=1|\mathbf{x}) = \frac{1}{1 + \exp(-h(\mathbf{x}))}$$

我们要算的权重 $\beta$ 是概率的比值：

$$\beta = \frac{P(z=1|\mathbf{x})}{P(z=-1|\mathbf{x})}$$

那既然目标是求比值，所以不需要再将原始打分变成概率，直接推导公式可以得到

$$\beta_i = \exp(h(\mathbf{x}_i))$$


**阶段二：正式计算**

- 数据输入:
训练集的特征 $X$、真实标签 $Y$，以及刚刚算好的身价表 $\beta$。

- 训练主角:
主任务模型 $f$ 登场。

- Loss 计算:
在算损失时，必须乘上各自的身价权重。模型被迫去“偏心”那些 $\beta$ 大的样本：
$$\text{Loss} = \frac{1}{n} \sum_{i=1}^{n} \beta_i \cdot l(f(x_i), y_i)$$

**核心差异总结**

| 维度 | 传统过程 | 两段式协变量纠正 |
| :--- | :--- | :--- |
| **Pipeline 架构** | 单一阶段 | 两个阶段（先算权重，再正式训练） |
| **计算量代价** | 正常主模型算力 | 主模型算力 + 极少量的轻分类器算力 |
| **Loss 函数** | 平均相加 | 加权相加 |

**标签漂移**

与协变量漂移的影响顺序相反

**源数据分布 $q(y)$ (我们已知的训练集)**

假设你的训练集里有 1000 张片子，标签分布如下：
* 健康 (A): 500张 $\rightarrow q(y_A) = 0.5$
* 轻度 (B): 300张 $\rightarrow q(y_B) = 0.3$
* 重度 (C): 200张 $\rightarrow q(y_C) = 0.2$

**验证集上的混淆矩阵 $\mathbf{C}$**

训练好了一个多层感知机 (MLP)。我们在验证集上测试它，并统计出了混淆矩阵 $\mathbf{C}$。
*注意：混淆矩阵的每一列代表“真实的类别”，每一行代表“模型预测的类别”。列的总和必须为 1。*

假设统计出来的矩阵 $\mathbf{C}$ 是这样的：

| | 真实为 A (健康) | 真实为 B (轻度) | 真实为 C (重度) |
| :--- | :--- | :--- | :--- |
| **预测为 A** | 0.80 | 0.10 | 0.05 |
| **预测为 B** | 0.15 | 0.70 | 0.10 |
| **预测为 C** | 0.05 | 0.20 | 0.85 |

*(如果一个样本真实是健康 A，模型有 80% 的概率猜对，15% 猜成轻度 B，5% 猜成重度 C)*

**测试集上的平均预测 $\mu(\hat{\mathbf{y}})$ (目标数据实测)**

现在，我们把这个模型部署到了一家新医院（目标域测试集）。这里所有的片子都**没有标签**。
你把所有片子输入模型，把模型输出的 Softmax 概率全部加起来求平均。假设我们得到了这样的总体预测结果：

* 模型总体预测为 A 的比例：$`22.5\% \rightarrow \mu(\hat{y}_A) = 0.225`$
* 模型总体预测为 B 的比例：$`41.0\% \rightarrow \mu(\hat{y}_B) = 0.410`$
* 模型总体预测为 C 的比例：$`36.5\% \rightarrow \mu(\hat{y}_C) = 0.365`$

**求解真实的测试集分布 $`p(\mathbf{y})`$**

现在我们要用到核心公式：$`\mathbf{C} p(\mathbf{y}) = \mu(\hat{\mathbf{y}})`$。
写成矩阵形式就是：

```math
\begin{bmatrix}
0.80 & 0.10 & 0.05 \\
0.15 & 0.70 & 0.10 \\
0.05 & 0.20 & 0.85
\end{bmatrix}
\begin{bmatrix}
p(y_A) \\
p(y_B) \\
p(y_C)
\end{bmatrix}
=
\begin{bmatrix}
0.225 \\
0.410 \\
0.365
\end{bmatrix}
```

通过矩阵求逆运算（$`\mathbf{C}^{-1}`$ 乘以右边的向量），我们可以解出真实的测试集分布 $p(\mathbf{y})$：
* $`p(y_A) = 0.20`$ (真实健康比例只有 20%)
* $`p(y_B) = 0.50`$ (真实轻度比例高达 50%)
* $`p(y_C) = 0.30`$ (真实重度比例为 30%)

**计算纠正权重 $\boldsymbol{\beta}$ 并重训**

现在我们有了测试集的真实估计 $p(\mathbf{y})$，结合训练集的 $q(\mathbf{y})$，计算权重 $\beta = p(y) / q(y)$：

* 健康 (A) 的权重： $`\beta_A = 0.20 / 0.50 = 0.4`$
* 轻度 (B) 的权重： $`\beta_B = 0.50 / 0.30 \approx 1.67`$
* 重度 (C) 的权重： $`\beta_C = 0.30 / 0.20 = 1.5`$

**代码**

在使用 PyTorch 训练 MLP 时，只需要在定义交叉熵损失函数时，传入这个权重向量即可：

```python

# 定义算出的权重向量 [0.4, 1.67, 1.5]
weights = torch.tensor([0.4, 1.67, 1.5], dtype=torch.float32) 

# 将权重传入损失函数
criterion = nn.CrossEntropyLoss(weight=weights)

# 之后的训练循环中，损失计算就会自动带上标签偏移修正
# loss = criterion(outputs, labels) 
```

自定义的块
```
import torch
import torch.nn as nn
# 定义“块” (把基础层打包)
class MySimpleBlock(nn.Module):
    def __init__(self, in_size, out_size):
        super().__init__()
        # 在块的内部，采购基础零件
        self.layer1 = nn.Linear(in_size, out_size)
        self.relu = nn.ReLU()
        self.layer2 = nn.Linear(out_size, out_size)

    def forward(self, x):
        # 定义块内部的流水线
        x = self.layer1(x)
        x = self.relu(x)
        x = self.layer2(x)
        return x
```
官方给的顺序块
```
# PyTorch 官方的 nn.Sequential 内部逻辑
class Sequential(nn.Module):
    def __init__(self, *args):
        # *args表示任意多的参数
        # ... 这里负责把传进来的层存进一个字典里 ...
        pass
        
    def forward(self, x):
        # 它自动写了一个 for 循环，遍历塞进来的所有零件
        for module in self._modules.values():
            x = module(x)  # 上一个层的输出 x，自动变成了下一个层的输入 x
        return x
```

**从全连接到卷积**

两个原则:
- 平移不变性:在不同位置，要找的东西不会发生巨大的特征改变
- 局部性:找一个目标不需要看得太远,看局部的信息即可

---
**新的表示形式**

之前做全连接，是把图片拍平，做成一维向量。现在要把它还原成矩阵，因为要考虑空间的信息(比如平移)

传统的一维全连接层 (1D MLP)：为了方便对比，我们不写成矩阵相乘的形式 $`\mathbf{h} = \mathbf{W}\mathbf{x} + \mathbf{b}`$，而是写成标量（单个元素）带下标的形式。假设我们要计算隐藏层中的第 $`i`$ 个神经元的值 $`h_i`$(对于黑白图来说,它是数字,如果是彩色图RGB,则是一个向量)：

$$h_i = b_i + \sum_{j} W_{i,j} x_j$$

保留空间结构的二维全连接层 (D2L 教材公式)：我们要计算隐藏层二维矩阵中，位于第 $`i`$ 行、第 $`j`$ 列的那个神经元的值 $`[\mathbf{H}]_{i,j}`$：

$$[\mathbf{H}]_{i,j} = [\mathbf{U}]_{i,j} + \sum_{k} \sum_{l} [\mathsf{W}]_{i,j,k,l} [\mathbf{X}]_{k,l}$$

| 组成部分 | 传统 1D 公式 | 教材 2D 公式 | 物理意义解释 |
| :--- | :--- | :--- | :--- |
| **目标神经元** | $`h_i`$ | $`[\mathbf{H}]_{i,j}`$ | 隐藏层中正在计算的那一个神经元的值。以前通过1个坐标 $(i)$ 就能找到它，现在需要2个坐标 $`(i, j)`$。 |
| **偏置项 (Bias)** | $`b_i`$ | $`[\mathbf{U}]_{i,j}`$ | 专属这个神经元的固定偏差值。 |
| **输入来源** | $`x_j`$ | $`[\mathbf{X}]_{k,l}`$ | 输入层中的某一个像素（特征）值。以前在向量里用 $`j`$ 找，现在在图像里用 $`(k, l)`$ 找。 |
| **“全连接”动作** | $`\sum_{j}`$ | $`\sum_{k} \sum_{l}`$ | “收集所有信息”。一维时遍历整条线（所有 $`j`$）；二维时遍历整个面（所有行 $`k`$ 和所有列 $`l`$）。 |
| **连接权重** | $`W_{i,j}`$ | $`[\mathsf{W}]_{i,j,k,l}`$ | （最核心的区别）连接“输入来源”与“目标神经元”的这根线上的权重乘数。 |

**为什么权重从二维矩阵变成了四阶张量？**

权重的维度，取决于你需要多少个坐标来定位连线的两头。

在传统 1D 结构中：定位输出节点需要 $1$ 个坐标（第 $`i`$ 个）。定位输入节点需要 $1$ 个坐标（第 $`j`$ 个）。$`1 + 1 = 2`$。所以权重 $`W_{i,j}`$ 只需要 $2$ 个下标，它是一个二维矩阵。

在保留空间结构的 2D 结构中：定位输出节点（隐藏层的某一点）需要 $2$ 个坐标（第 $`i`$ 行，第 $`j`$ 列）。定位输入节点（原图的某一点）也需要 $2$ 个坐标（第 $`k`$ 行，第 $`l`$ 列）。$`2 + 2 = 4`$。所以权重 $`[\mathsf{W}]_{i,j,k,l}`$ 必须要有 $`4`$ 个下标，它变成了一个巨大的四阶张量。

---
**平移不变性**

$`\mathbf{X}`$的平移导致$`\mathbf{H}`$的平移,但又不应该影响到$`\mathbf{V}`$和$`\mathbf{U}`$，因此要加一个限制(平移不变性假设)，使得四阶张量$`[\mathsf{V}]_{i,j,a,b}`$成为二位矩阵$`[\mathsf{V}]_{a,b}`$,$`u`$也成为一个常数,从而公式瘦身成为:

$$[\mathbf{H}]_{i,j} = u + \sum_a \sum_b [\mathbf{V}]_{a,b} [\mathbf{X}]_{i+a,j+b}$$

那个剩下的二维矩阵 $`[\mathbf{V}]_{a,b}`$，就是我们常说的卷积核 (Kernel) 或 滤波器 (Filter)。

网络不再为每一个像素坐标分配专属的权重，而是拿着这唯一的一组权重 $`[\mathbf{V}]_{a,b}`$（卷积核），像盖章一样，在整张图像的各个 $`(i, j)`$ 位置上滑动，进行相同的计算。

**局部性**

进一步地,为了收集用来训练参数 $`[\mathbf{H}]_{i,j}`$ 的相关信息，我们不应偏离到距 $`(i, j)`$ 很远的地方。这意味着在 $`|a| > \Delta`$ 或 $`|b| > \Delta`$ 的范围之外，我们可以设置 $`[\mathbf{V}]_{a,b} = 0`$。因此，我们可以将 $`[\mathbf{H}]_{i,j}`$ 重写为

$$[\mathbf{H}]_{i,j} = u + \sum_{a=-\Delta}^{\Delta} \sum_{b=-\Delta}^{\Delta} [\mathbf{V}]_{a,b}[\mathbf{X}]_{i+a,j+b}$$

至此,完成了全连接层到卷积层的转化,卷积层也是特殊的全连接层.

**为什么称之为卷积？**

在数学中，两个函数（比如 $`f,g:\mathbb{R}^d\rightarrow\mathbb{R}`$）之间的“卷积”被定义为

$$(f * g)(\mathbf{x}) = \int f(\mathbf{z})g(\mathbf{x} - \mathbf{z})d\mathbf{z}$$

卷积是当把一个函数“翻转”并移位 $\mathbf{x}$ 时，测量 $f$ 和 $g$ 之间的重叠.离散形式,积分变成求和,就可以找到对应.

**变化**

旧公式(单通道灰度图)：

$$[\mathbf{H}]_{i,j} = \sum_{a} \sum_{b} [\mathbf{V}]_{a,b}[\mathbf{X}]_{i+a,j+b}$$

(注：为了对比清晰，这里先忽略偏置 $u$)

新公式：

$$[\mathbf{H}]_{i,j,d} = \sum_{a} \sum_{b} \sum_{c} [\mathsf{V}]_{a,b,c,d}[\mathbf{X}]_{i+a,j+b,c}$$

对比之下,发生了4个核心变化: 

1. 输入 $`\mathbf{X}`$ 长厚了：多了一个坐标 $`c`$以前的 $`[\mathbf{X}]_{i,j}`$ 只是一个单层矩阵（黑白图片）。现在的 $`[\mathbf{X}]_{i+a,j+b,c}`$ 变成了一个三维张量（立体方块）。新增的 $`c`$ 代表输入通道 (Channel)。比如对于一张普通的彩色照片，$`c`$ 可以取 3 个值：红(R)、绿(G)、蓝(B)。这意味着输入变成了一个有三层厚度的“三明治”。

2. 输出 $`\mathbf{H}`$ 也长厚了：多了一个坐标 $`d`$以前的 $`[\mathbf{H}]_{i,j}`$ 是单层结果。现在的 $`[\mathbf{H}]_{i,j,d}`$ 也变成了立体的。新增的 $`d`$ 代表输出通道 (特征映射 Feature Maps)。
    - 为什么要多个输出通道？因为在一个位置 $`(i,j)`$，我们不仅想知道“这里有没有横向边缘”，还想同时知道“这里有没有竖向边缘”、“这里有没有猫的胡须纹理”。
    - 每个输出通道 $`d`$ 都负责专门提取一种特定的特征。

3. 连加符号 $`\sum`$ 变多了：增加了 $`\sum_c`$以前只在二维平面的宽度和高度 $`(a, b)`$ 上滑动求和。现在多了一个 $`\sum_c`$。这意味着，当我们算出一个输出点时，不仅要把空间上 $`3 \times 3`$ 范围内的像素加起来，还要把所有输入通道（红、绿、蓝层）在同一位置的信息一起加起来。
    - 物理直觉：我们在看一朵红花时，我们的大脑是同时综合了它在红色层的高亮度和绿色层的低亮度，才判定它是红色的。所以不同输入层的数据必须在这一步融合。

4. 卷积核 $`\mathsf{V}`$ 变成了四维：$`[\mathsf{V}]_{a,b,c,d}`$这是最烧脑的一点。原本的二维小印章 $`[\mathbf{V}]_{a,b}`$ 不够用了，它进化成了包含了 $`a,b,c,d`$ 四个维度的四阶张量。
    - $`a, b`$：决定了宽和高（比如 $`3 \times 3`$）。
    - $`c`$：决定了它要看输入图像的哪一层（红、绿还是蓝）。为了匹配输入图像，卷积核自身也必须变成有厚度的“立体方块”。
    - $`d`$：决定了它要把提取到的特征写到输出层 $`\mathbf{H}`$ 的哪一页。

- 核矩阵大小是超参数

训练
```
# 1. 实例化我们刚才定义的模型
conv2d = Conv2D(kernel_size=(1, 2))

# 2. 定义一个损失函数（比如均方误差）和一个优化器（比如梯度下降）
# 优化器需要知道它要更新哪些参数，所以我们把 conv2d.parameters() 传给它
learning_rate = 0.03

# 3. 开始“训练循环” (这里假设迭代 10 次)
for i in range(10):
    # 【正向传播】调用上面类里的 forward 函数，算出预测值
    Y_hat = conv2d(X)
    
    # 【计算损失】对比预测值和真实值 Y，看看差了多少
    loss = (Y_hat - Y) ** 2 
    
    # 【反向传播】这一步最关键！框架会自动算出 loss 对 weight 和 bias 的梯度
    conv2d.zero_grad() # 清空上一轮的梯度
    loss.sum().backward() # 自动求导
    
    # 【更新参数】根据梯度，更新我们积木里的 weight 和 bias
    conv2d.weight.data[:] -= learning_rate * conv2d.weight.grad
    conv2d.bias.data[:] -= learning_rate * conv2d.bias.grad
    
    if (i + 1) % 2 == 0:
        print(f'epoch {i+1}, loss {loss.sum().item():.3f}')
```

填充(Padding)需要考虑奇偶，以解决非对称导致的图像漂移问题

当满足：
- 卷积核的大小是奇数；
- 所有边的填充行数和列数相同；
- 输出与输入具有相同高度和宽度

则可以得出：输出Y[i, j]是
通过以输入X[i, j]为中心，与卷积核进行互相关计算得到的

---
多通道(比如彩色图RGB,有三个通道)输入，其核函数同样也是多通道的(一一对应)，而一个特征往往需要多个通道来共同定义，所以不同通道的结果被强行加在了一起。

**每个输出通道，都可以是一个人为指定的特征？**

在旧时代，可能需要专家写一个核，去提取垂直，边缘特征。而在现代的 CNN， 
当我们写下代码 out_channels=64 时，我们仅仅是给网络分配了 64 个空白的、装满随机数字的卷积核。我们绝对不会去告诉第 1 个核“你去学猫耳朵”，告诉第 2 个核“你去学猫尾巴”。

**那网络是怎么知道这 64 个通道该学什么的呢？**

全靠损失函数（Loss）和反向传播（梯度下降）。

- 网络用随机的 64 个核扫过图片，得出一个瞎猜的结果（比如把猫认成了狗）。

- 我们告诉网络：“你认错了，误差（Loss）很大！”

- 网络通过数学求导发现：“哦，如果我把第 12 号卷积核里的数字稍微改一下，让它对尖尖的形状（猫耳朵）敏感一点，我的总误差就能变小。”

- 经过成千上万次的迭代，网络为了拼命降低误差，硬生生地把这 64 个原本随机的核，逼成了专门提取各种有用特征的“专家”。

所以，CNN是个"黑盒"。特征多样性导致了一个输出通道需要配一个专属的核

---
**用卷积层替代全连接权重矩阵来进行前期的特征提取，用极少的参数量完美保留了图像的二维空间结构**
既然如此，那卷积层的核函数该如何更新？

**第一步：谁是需要被更新的“参数”？**

在多层感知机中，需要更新的是巨大的权重矩阵 $`W`$。在卷积层中，需要被更新的，仅仅是那个 $`3 \times 3`$ 卷积核里的 9 个具体的数字（以及 1 个偏置 $`u`$）。假设一开始，这 9 个数字是随机生成的瞎猜值。

**第二步：前向传播与计算 Loss**

- 这个随机的 $`3 \times 3`$ 卷积核在输入图像 $`X`$ 上滑动，得出一张输出特征图 $`Y`$。
- 特征图经过后面的层（展平、全连接），最终输出一个预测结果（比如“认定这是一只狗”）。
- 我们比对真实标签（“其实这是一只猫”），计算出巨大的误差 Loss。第三步：反向传播（见证奇迹的时刻）在反向传播时，我们要计算 Loss 对这 9 个参数的导数（梯度），看看它们该往哪个方向修改。


**第三步：反向传播**

这里出现了和全连接层最大的不同:
- 在全连接层中：一个权重 $`w_{i,j}`$ 只连接一个输入和一个输出，所以它挨骂（背锅）也只背一次。
- 在卷积层中：因为权重共享，核里面的某一个参数（比如左上角的权重 $`v_{0,0}`$），在前向传播时像印章一样，盖满了整张图像的每一个角落，参与了成百上千次计算

既然左上角的 $`v_{0,0}`$ 参与了特征图上每一个像素的计算，那么当计算 $`v_{0,0}`$ 的梯度时，就必须把特征图上每一个像素传回来的误差，全部加起来.

用一句数学公式简单概括这个过程的链式法则：

$$\frac{\partial \text{Loss}}{\partial \text{权重}v} = \sum_{\text{所有滑过的位置}} \left( \text{对应位置的输入} X \times \text{对应位置传回来的误差梯度} \right)$$

一旦算出了这 9 个参数各自的综合梯度，最后一步就是我们熟悉的梯度下降公式了：

$$v_{\text{新}} = v_{\text{旧}} - \text{学习率} \times \text{梯度}$$
